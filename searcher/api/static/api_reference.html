Searcher API Reference

Scholarly search, web content extraction, and PDF download service. All endpoints return JSON. Served live at /api-reference.

Prod https://searcher.xds-lab.com/aev/search
 
Local http://127.0.0.1:8000

Interactive OpenAPI docs: <base>/docs

Response Format

Search Envelope

All search endpoints wrap results in a common envelope.

{
  "provider":                "openalex",
  "query":                   "UAV trajectory optimization",
  "search_date":             "2026-05-27",
  "total_records_available": 69114,   // null for browser-scraped sources
  "total_records_downloaded": 200,
  "results": [ ... ]
}

Result Record

Every item in results uses this shape. Empty string when a field is unavailable. authors is always an array. is_abstract is true when snippet contains a full abstract.

{
  "title":            "Energy-Efficient UAV Communication With Trajectory Optimization",
  "url":              "https://doi.org/10.1109/twc.2017.2688328",
  "publication_year": 2017,
  "authors":          ["Yong Zeng", "Rui Zhang"],
  "doi":              "10.1109/twc.2017.2688328",
  "source":           "IEEE Transactions on Wireless Communications",
  "snippet":          "Wireless communication with unmanned aerial vehicles (UAVs) is a promising...",
  "is_abstract":      true,
  "pdf_link":         "https://arxiv.org/pdf/1608.04792"
}

Abstract Coverage

The snippet field carries the abstract text when available. is_abstract: true indicates a full abstract. is_abstract: false means only a short excerpt or nothing.

EndpointAbstract?Notes
/search_openalexAlwaysFull abstract reconstructed from OpenAlex inverted index
/search_semantic_scholarAlwaysFull abstract from Semantic Scholar API
/search_ieeexploreAlwaysFull abstract from IEEE Xplore API
/search_web_of_scienceUsuallyFrom BibTeX export; included when WoS exports it
/search_scopusDefaultFetched via Abstract Retrieval API (1 extra call/result). Set include_abstract=false to skip.
/search_sciencedirectPartialShort dc:description from search API — not always a full abstract
/search_ebscoPartialScraped excerpt when visible on results page
/search_google_scholar_browserExcerpt2–3 sentence Google Scholar snippet only

Environment Variables

Configure via .env (production) or .env.dev (local). Copy .env.example to get started.

VariableUsed byRequired?Description
IEEE_XPLORE_API_KEY /search_ieeexplore Yes IEEE Xplore Metadata API key
ELSEVIER_API_KEY /search_scopus, /search_sciencedirect Yes Elsevier API key (Scopus + ScienceDirect + Abstract Retrieval)
OPENALEX_API_KEY /search_openalex Optional OpenAlex API key. Unauthenticated access is allowed; key raises rate limits.
SEMANTIC_SCHOLAR_API_KEY /search_semantic_scholar Optional Raises rate limits when set
BROWSER_WORKER_URL All browser-backed endpoints Yes Internal URL of the browser_worker service (default: http://127.0.0.1:8010)
REQUEST_TIMEOUT_SECONDS All HTTP endpoints Optional Per-request timeout in seconds (default: 20)
PDF_MAX_MB /download_pdf Optional Maximum PDF size in MB (default: 50)
DOWNLOAD_DIR /download_pdf Optional Server-side directory where PDFs are saved (default: downloads/ at repo root)
USER_AGENT All HTTP endpoints Optional HTTP User-Agent header (default: searcher/1.0 (+https://localhost))

Utility Endpoints

GET
/health
Returns service status and current version string.

No parameters

Example response

{ "status": "ok", "version_name": "searcher-stack-v1.1.168" }
GET
/fetch_page
Fetch a URL and extract readable text, title, meta description, and up to 25 links.

Parameters

NameTypeDefaultDescription
url* string— Page URL to fetch
include_html bool false Include raw HTML in the response
max_chars int ≥ 500 12000 Maximum characters of body text to return

Example response

{
  "url":              "https://ieeexplore.ieee.org/document/8714567/",
  "title":            "Page title",
  "meta_description": "Meta description",
  "text":             "Extracted body text...",
  "links":            ["https://...", "...up to 25 hrefs"]
}
GET
/review_page
Like /fetch_page but adds word count, estimated read time, and heading list — suited for AI review tasks.

Parameters

NameTypeDefaultDescription
url* string— Page URL to fetch
include_html bool false Include raw HTML in the response
max_chars int ≥ 500 12000 Maximum characters of body text to return

Additional response fields

{
  "word_count":                  3200,
  "estimated_read_time_minutes": 15,
  "headings":                    ["Introduction", "Related Work", "...up to 15 h1/h2/h3"]
}
GET
/download_pdf
Stream a PDF from a direct URL and save it server-side. Does not use the browser session.
max 50 MB by default

Parameters

NameTypeDefaultDescription
url* string— Direct URL to a PDF file

Example response

{ "path": "/opt/searcher/downloads/2104.02239.pdf", "filename": "2104.02239.pdf", "size_bytes": 843210 }

Error codes

  • 400 — URL does not appear to be a PDF
  • 413 — PDF exceeds PDF_MAX_MB
  • 502 — Fetch failed
  • For subscription PDFs use the browser_worker /download_paper endpoint instead

Search Endpoints

GET
/search_openalex
Search OpenAlex — a free, open index of 250M+ scholarly works. Full abstracts always included. Cursor-based pagination, up to 10,000 results per query.
No key required Abstracts always included

Parameters

NameTypeDefaultDescription
query* string— Full-text search query. Supports Boolean operators.
limit int ≥ 1 200 Maximum results to return. Fetched in pages of 100. Max 10,000.
year_low int null Earliest publication year (inclusive)
year_high int null Latest publication year (inclusive)
is_oa bool null true restricts to open access works only. Open access results include pdf_link.
work_type string null Filter by work type: article, book, dataset, preprint, review, etc.

Examples

# 500 results with abstracts
GET /search_openalex?query=UAV+trajectory+optimization&limit=500

# Open access only, from 2020
GET /search_openalex?query=UAV+trajectory&limit=200&year_low=2020&is_oa=true

Notes

  • Abstracts are reconstructed from OpenAlex's inverted index format — leading "Abstract" header is stripped automatically
  • Set OPENALEX_API_KEY to raise rate limits beyond 100k requests/day
  • Pagination is automatic cursor-based — just increase limit
GET
/search_semantic_scholar
Search Semantic Scholar via their public API. Good for CS/STEM coverage with citation data. Full abstracts always included.
No key required Abstracts always included

Parameters

NameTypeDefaultDescription
query* string— Free-text search query
limit int ≥ 1 200 Maximum results to return

Example

GET /search_semantic_scholar?query=UAV+trajectory+optimization&limit=50

Notes

  • Does not support Boolean operators or year filtering
  • Set SEMANTIC_SCHOLAR_API_KEY for higher rate limits
GET
/search_ieeexplore
Search IEEE Xplore via the IEEE Xplore Metadata API. Supports Boolean operators and rich filtering. Auto-paginates in pages of 200.
Requires IEEE_XPLORE_API_KEY Abstracts always included 1-based pagination

Parameters

NameTypeDefaultDescription
query* string— Search query. Supports AND, OR, NOT
limit int ≥ 1 200 Maximum results to return. Auto-paginates in batches of 200.
start_record int ≥ 1 1 1-based offset. Use for manual pagination.
year_low int null Earliest publication year (inclusive)
year_high int null Latest publication year (inclusive)
content_type string null Books, Conferences, Courses, Early Access, Journals, Magazines, Standards
open_access bool null true restricts to open-access articles only
sort_field string null article_number, article_title, or publication_title
sort_order string null asc or desc
author string (min 3) null Filter by author first or last name (minimum 3 characters)

Examples

# Journal search with year filter
GET /search_ieeexplore?query=UAV+AND+trajectory&limit=200&year_low=2018&content_type=Journals

# Manual pagination — records 201–400
GET /search_ieeexplore?query=UAV+AND+trajectory&limit=200&start_record=201

# Filter by author
GET /search_ieeexplore?query=UAV+trajectory&author=Zeng
GET
/search_scopus
Search Scopus (Elsevier) via the Scopus Search API. Supports full Scopus field-code Boolean syntax. Abstracts fetched by default via a separate API call per result.
Requires ELSEVIER_API_KEY Abstracts by default 0-based pagination

Parameters

NameTypeDefaultDescription
query* string— Scopus query. Field codes: TITLE(), ABS(), AUTH(), TITLE-ABS-KEY(), etc.
limit int ≥ 1 200 Maximum results to return. Fetched in pages of 25.
start int ≥ 0 0 0-based result offset
year_low int null Earliest publication year (inclusive)
year_high int null Latest publication year (inclusive)
subj string ENGI Subject area code. Common: ENGI, COMP, MATE, PHYS, MATH. Default ENGI is always applied unless overridden.
include_abstract bool true Fetch full abstract via Abstract Retrieval API (one extra API call per result). Set to false for faster bulk retrieval without abstracts.

Examples

# Field-code query with abstracts
GET /search_scopus?query=TITLE-ABS-KEY(UAV+trajectory)&limit=50&year_low=2018

# Fast bulk retrieval without abstracts
GET /search_scopus?query=TITLE-ABS-KEY(UAV)&limit=200&include_abstract=false

Notes

  • include_abstract=true makes one extra Elsevier API call per result — consumes Abstract Retrieval API quota (typically 20,000 requests/week per key)
  • Use small limits when fetching abstracts to avoid quota exhaustion
GET
/search_sciencedirect
Search ScienceDirect (Elsevier) full-text journals. Complements Scopus — covers Elsevier articles specifically.
Requires ELSEVIER_API_KEY Partial abstracts 0-based pagination

Parameters

NameTypeDefaultDescription
query* string— Free-text search query
limit int ≥ 1 200 Maximum results to return
start int ≥ 0 0 0-based result offset
year_low int null Earliest publication year (inclusive)
year_high int null Latest publication year (inclusive)

Example

GET /search_sciencedirect?query=UAV+trajectory+optimization&limit=50&year_low=2019
GET
/search_google_scholar_browser
Search Google Scholar by driving the persistent Chromium browser. Broad coverage including preprints and grey literature. Handles CAPTCHAs via noVNC.
Requires browser_worker Excerpt only (no full abstract) 0-based pagination

Parameters

NameTypeDefaultDescription
query* string— Search query
limit int ≥ 1 200 Maximum results to return
start_index int ≥ 0 0 0-based result offset
year_low int null Earliest publication year (inclusive)
year_high int null Latest publication year (inclusive)
exclude_domains list[string] null Domains to exclude from results. Default excludes researchgate.net, books.google.com, search.proquest.com. Pass empty list to disable.

Example

GET /search_google_scholar_browser?query=UAV+path+planning&limit=100&year_low=2020

Notes

  • total_records_available is always null — Google Scholar does not expose a total count
  • snippet contains a 2–3 sentence excerpt only; is_abstract is false
  • If a CAPTCHA appears, open noVNC, solve it once — the browser session persists
GET
/search_web_of_science
Search Web of Science by driving the persistent Chromium browser. Exports results as BibTeX. Abstracts included when WoS exports them.
Requires browser_worker + WoS session Abstracts usually included

Parameters

NameTypeDefaultDescription
query* string— Search query
limit int ≥ 1 200 Maximum results to return
year_low int null Earliest publication year (inclusive)
year_high int null Latest publication year (inclusive)

Example

GET /search_web_of_science?query=UAV+trajectory+optimization&limit=50&year_low=2019

Notes

  • Log in to Web of Science via noVNC before use if the session has expired
GET
/search_ebsco
Search EBSCO Research by driving the persistent Chromium browser. Requires an active institutional EBSCO session.
Requires browser_worker + EBSCO session Partial abstracts

Parameters

NameTypeDefaultDescription
query* string— Search query
limit int ≥ 1 200 Maximum results to return
year_low int null Earliest publication year (inclusive)
year_high int null Latest publication year (inclusive)

Example

GET /search_ebsco?query=UAV+path+planning&limit=100&year_low=2020

Notes

  • If access is blocked by a login wall, open noVNC, log in through the institution portal, then retry

EBSCO Download Endpoints

GET
/download_ebsco_paper
Download a single paper from an EBSCO detail page URL using the browser.
Requires browser_worker + EBSCO session

Parameters

NameTypeDefaultDescription
url* string— EBSCO paper detail page URL (e.g. https://research.ebsco.com/c/.../search/details/...)

Example

GET /download_ebsco_paper?url=https://research.ebsco.com/c/abc123/search/details/xyz789

Response

{ "path": "...", "filename": "...", "size_bytes": 843210 }
POST
/download_ebsco_papers
Download multiple EBSCO papers sequentially from a list of detail page URLs.
Requires browser_worker + EBSCO session

Request body

JSON array of EBSCO detail page URL strings.

[
  "https://research.ebsco.com/c/.../search/details/url1",
  "https://research.ebsco.com/c/.../search/details/url2"
]

Notes

  • Each URL is processed in order; a failure on one does not stop the batch
  • Request timeout scales with URL count: 60s × n + 30s

ATRD Endpoints

GET
/search_atrd_papers
Return metadata for all papers from the ATRD 1st Symposium page. No file downloads.
No auth required No browser required

Parameters

None. The source URL is fixed: https://www.atrdsymposium.org/past-seminars/1st-symposium/papers-and-presentations/

Response

{
  "source": "https://www.atrdsymposium.org/...",
  "count": 53,
  "papers": [
    {
      "title": "Vertiport Placement for Urban Air Mobility to Reduce Time for Multimodal Travel",
      "authors": ["Yashovardhan S. Chati", "Yamini Sharma", "Kirti Nikam", "Prasad Devkar", "Arunchandar Vasan"],
      "section": "Autonomous, unmanned and remotely piloted aircraft systems and emerging operations",
      "is_best_paper": true,
      "full_paper_url": "https://drive.google.com/file/d/FILE_ID/view?usp=sharing",
      "presentation_url": "https://drive.google.com/file/d/FILE_ID/view?usp=sharing"
    }
  ]
}

Notes

  • section — the symposium topic category the paper belongs to
  • is_best_paper — true when the paper received a Best Paper award
  • presentation_url — empty string for doctoral papers (slides not posted)
GET
/download_atrd_papers
Scrape the ATRD 1st Symposium papers page, download all full-paper PDFs from Google Drive, and return structured metadata.
Playwright (headless) No auth required

Parameters

None. The source URL is fixed: https://www.atrdsymposium.org/past-seminars/1st-symposium/papers-and-presentations/

Response

{
  "papers": [
    {
      "title": "Vertiport Placement for Urban Air Mobility",
      "authors": ["Jane Doe", "John Smith"],
      "url": "https://drive.google.com/file/d/FILE_ID/view?usp=sharing",
      "local_path": "/opt/searcher/downloads/atrd/vertiport_placement_for_urban_air_mobility.pdf",
      "size_bytes": 843210
    }
  ],
  "count": 42,
  "errors": []
}

Notes

  • Uses a headless Chromium browser — page is JavaScript-rendered
  • PDFs are saved under <DOWNLOAD_DIR>/atrd/; re-running skips files already on disk
  • Each errors entry describes a paper that could not be downloaded (bad link, parse failure, etc.) without stopping the batch

Common Patterns for AI Agents

Which search endpoint should I use?

GoalRecommended endpoint(s)
Full abstracts, free, broadest coverage/search_openalex
Peer-reviewed engineering / CS with abstracts/search_ieeexplore, /search_semantic_scholar
Broad Scopus coverage with abstracts/search_scopus (default include_abstract=true)
Elsevier full-text articles/search_sciencedirect
Broad coverage incl. preprints & grey lit/search_google_scholar_browser
WoS citation tracking/search_web_of_science
EBSCO database access/search_ebsco

Check whether a result has a full abstract

# is_abstract=true means snippet contains a full abstract
if result["is_abstract"]:
    abstract = result["snippet"]

Fetch full page content from a result

GET /fetch_page?url={result.url}
# Or for AI review tasks:
GET /review_page?url={result.url}

Download a paper

# Open-access direct PDF link:
GET /download_pdf?url={result.pdf_link}

# Subscription paper (IEEE, Elsevier, etc.) — use browser_worker:
POST https://searcher.xds-lab.com/aev/browser/download_paper
Content-Type: application/json
{ "url": "{result.url}" }

Manual pagination

# IEEE — 1-based start_record
GET /search_ieeexplore?query=UAV&limit=200&start_record=1
GET /search_ieeexplore?query=UAV&limit=200&start_record=201

# Scopus / ScienceDirect — 0-based start
GET /search_scopus?query=UAV&limit=25&start=0
GET /search_scopus?query=UAV&limit=25&start=25

# Google Scholar browser — 0-based start_index
GET /search_google_scholar_browser?query=UAV&limit=10&start_index=0
GET /search_google_scholar_browser?query=UAV&limit=10&start_index=10

# OpenAlex — automatic cursor pagination, just increase limit
GET /search_openalex?query=UAV&limit=1000
